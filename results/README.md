# results/

Working directory for training and evaluation outputs. Each run writes here
(configurable with `--output-dir`):

- `best_model.pt` — best checkpoint (by validation accuracy). Not committed (large).
- `config.json` — the exact `Config` used, so evaluation reproduces the split.
- `class_names.txt` — class order.
- `metrics.json` — test accuracy, precision/recall/F1 (macro & weighted), per-class report.
- `training_curves.png`, `confusion_matrix.png`, `baseline_comparison.png` — plots.

The contents are git-ignored (they are regenerated by the scripts). A committed
snapshot of the plots and metrics from the synthetic-data demo lives in
[`../docs/demo_results/`](../docs/demo_results/).
